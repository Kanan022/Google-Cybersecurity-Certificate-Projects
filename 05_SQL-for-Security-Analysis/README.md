## Project Summary
A demonstration of using **Structured Query Language (SQL)** to investigate security incidents and audit user data. This involved writing queries to filter for specific security events in a database (e.g., failed login attempts, specific timeframes) and identify non-compliant users.

## Key Skills Demonstrated
* **Database Forensics:** Using SQL to query and extract security-relevant information from logs and user tables, a foundational skill for threat hunting.
* **SQL Querying:** Proficiency with core SQL filtering clauses: **`SELECT`**, **`WHERE`**, and logical operators **`AND`**, **`OR`**, and **`NOT`**.
* **Pattern Matching:** Utilizing the **`LIKE`** operator with wildcards (`%`) to find patterns in data fields.
* **Security Auditing:** Identifying groups of employees (e.g., those outside the IT department) who require security updates or specific policy enforcement.
